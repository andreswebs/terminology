# E4.T1 — ErrInvalidField sentinel

## Goal

Add the `ErrInvalidField` sentinel error to `internal/output/errors.go`. This sentinel is returned when `--fields` specifies an unknown dotted path. It's a usage error (exit 2) per the error-handling ADR, not a data error.

## Refs

- E4 spec: [docs/specs/004-read-commands.md](docs/specs/004-read-commands.md) §"--fields projection"
- Error-handling ADR: [docs/adr/error-handling.md](docs/adr/error-handling.md) — usage errors exit 2
- Schema-source-of-truth ADR: [docs/adr/schema-source-of-truth.md](docs/adr/schema-source-of-truth.md) — sentinel-as-registry

## Files to create / modify

- `src/internal/output/errors.go` — add `ErrInvalidField` sentinel
- `src/internal/output/errors_test.go` — test sentinel satisfies `terr.Coded`, correct code/exit/hint

## Behavior

```go
var ErrInvalidField = terr.New(
    "invalid_field", 2,
    "see `terminology schema --command CMD` for valid paths",
    "unknown field path",
)
```

- Code: `invalid_field`
- Exit: `2` (usage error)
- Hint: `see \`terminology schema --command CMD\` for valid paths`
- Message: `unknown field path`

The sentinel is consumed by T2 (--fields projection) to wrap specific path validation errors.
